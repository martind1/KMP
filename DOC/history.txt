14.02.98  LookUpEdit - References
          -----------------------
          Referenzierung einer Fremdtabelle über mehrere Key-Segmente
          wurden bisher wurden bisher über TLookUpEdit.References
          angegeben. Dies hatte den Nachteil, daß die Reihenfolge der
          Keysegmente nicht korrekt angegeben werden konnte, wenn das
          LookUpEdit-Feld nicht das erste Keysegment ist.

          Jetzt wird References nicht mehr benötigt.
          Die Reihenfolge und die Referenzen werden jetzt über die
          KeyList i.V.m. ->SOList ermittelt.
          Wichtig: LookUpDef.KeyList muß alle Mehr-Segment-Keys enthalten.

          SOList
          ~~~~~~
          Liste mit allen Synonym-Feldern (incl. Referenz-Felder).

14.02.98  LookUpDef - SOList
          ------------------
          Die Liste enthält die Synonym-Felder (SO-Felder) und ihre
          Referenzen zu der Lookup-Tabelle.
          Zum jetzigen Stand wird die Liste immer automatisch angelegt,
          indem die LookUpEdit-Felder ausgewertet werden (DataField,
          LookUpField).
          Sie wird für TLookUpDef.PutSOFields verwendet, um in
          NavLink.NewDataChange die Werte zu übernehmen.
          (bisher: TDataPos.PutMasterFields).
          Sie wird auch zum Nachladen aus einer LookUp-Tabelle nach
          Feldänderungen bzw. nach der Übernahme verwendet.
          (bisher: TLookUpEdit.BCDataChange).

14.02.98  Feldbeschreibungen zwischenspeichern
          ------------------------------------
          Beim ersten Öffnen einer Query wird die Feldbeschreibung
          zwischengespeichert und beim wiederholten Öffnen (auch
          einer anderen Query mit gleichem SQL) kopiert, so daß
          das u.U. langwierige Lesen entfällt.

          Intern:
          Die Speicherung erfolgt in der Stringliste
          GNavigator.FieldDesc; die Strings enthalten das SQL in
          komprimierter Form, die Objekte enthalten die FieldDefs.

          Permanent speichern:
          Die Feldbeschreibungen können permanent gespeichert und
          wieder eingelesen werden.

          Speichern in .INI: FieldDescSaveToIni;    (Modul: FldDeDlg.PAS)
          Laden von .INI: FieldDescLoadFromIni;
          Vosicht: Die einzelnen Zeilen können >1000 Zeichen lang sein !

          Dialog:
          GNavigator.FieldDescDlg;

          Aufrufparameter zum Laden:
          LoadFieldDesc=1

02.03.98  Löschen in Detailtabellen
          -------------------------
          1. DeleteDetails = false:
             - Statt zu löschen werden die Reference-Felder auf null gesetzt,
               d.h. nur die Zuordnung wird aufgehoben.
             - Dies kann zu Fehlern führen, wenn das Feld not null ist
neu >>>   2. DeleteDetails = true (oder LookUpSource <> nil):
             - Normales Löschen
             Anmerkung:
             DeleteDetails=true hat noch den Effekt, daß beim Löschen eines
             Datensatzes in der Haupttabelle auch alle referenzierten
             Datensätze der Lookup-Tabelle gelöscht werden.


02.03.98  Erfassen in Detailtabellen
          --------------------------
          1. DeleteDetails = false:
             - Es wird eine LookUp-Tabelle aufgeblendet mit noch nicht
               referenzierten Details (Reference-Felder = null).
          2. LookUpSource <> nil (n:m):
             - Es wird eine Tabelle des LookUpSources aufgeblendet.
               Die Übernahme erfolgt ohne daß die Tabelle weggeblendet wird.
               Bei Übernahme wird ein Datensatz in der Zuordnungstabelle neu
               angelegt mit den Werten der Reference-Felder bzgl. Haupttabelle
               und LookUpSource-Tabelle.
neu >>>   3. DeleteDetails = true
             - Normales Erfassen in der Tabelle
          3.3. DeleteDetails = true und ErfassSingle = true
               - Erfassen in der Fremdmaske (dort in der Einzesicht)
                 vergl. auch LNavigator.ErfassSingle
